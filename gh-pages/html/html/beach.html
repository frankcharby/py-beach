handle(requestType, handlerFunction)[source]¶

Initiates a callback for a specific type of request.

Parameters:
  • requestType – the string representing the type of request to handle
  • handlerFunction – the function that will handle those requests, this function receives a single parameter (the message) and returns a message to reply to the message. If it returns True, a generic success message will be replied, and if it returns a simple string, it will reply a generic error message where the string is the error message. To return data, return a dict.
Returns:

the previous handler for the request type or None if None existed

addActor(actorName, category, strategy='random', strategy_hint=None, realm=None)[source]¶

Spawn a new actor in the cluster.

Parameters:
  • actorName – the name of the actor to spawn
  • category – the category associated with this new actor
  • strategy – the strategy to use to decide where to spawn the new actor, currently supports: random
  • strategy_hint – a parameter to help choose a node, meaning depends on the strategy
  • realm – the realm to add the actor in, if different than main realm set
Returns:

returns the reply from the node indicating if the actor was created successfully, use beach.utils.isMessageSuccess( response ) to check for success
